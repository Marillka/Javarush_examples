Принцип единой ответственности.
При разработке необходимо проектировать класса так, чтобы объект выполнял одну цель. Тогда при внесении изменений в класс вы не будете оказывать влияние на другие объекты.

Например есть класс Entity (сущность), у которого есть методы получения имени, возраста, вставки, удаления и изменения. В этом классе задачи разбиваются на два типа: работа с сущностью и работа с хранилищем данны. Когда мы захотим поменять информацию о сущности, изменяем сам класс, а это может привести к его нерапотоспособности.

Лучший вариант - разделить класс на две части по задачам.
