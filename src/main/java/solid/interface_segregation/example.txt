Принцип разделения интерфейса.

Этот принцип говорит, что лучше не стоит создавать огромные интерфейсы, которые включают иного невостребованных методов. Лучше разделить такой интерфейс на несколько маленьких. В итоге при изменении интерфейса снижается вероятность, что необходимо будет вносить изменения в класс реализации.
